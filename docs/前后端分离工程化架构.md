# 前后端分离工程化架构

## 目标

把 `ai-manga-factory` 从“FastAPI 直接挂单文件前端”升级为“API 与前端工程分离、同时保留旧界面兜底”的工程化结构。

## 当前结构

### 后端

- 技术：FastAPI
- 目录：`backend/`
- 职责：
  - 任务创建与执行
  - 适配包接口
  - provider 用量与阶段模型策略
  - job 级结果沉淀

### 前端

- 技术：React + TypeScript + Vite
- 目录：`web/`
- 职责：
  - 展示项目控制台
  - 展示最近任务、当前步骤与关键产物
  - 展示适配包与 job 级证据绑定
  - 提供通用任务和适配包运行入口

### 兼容层

- 旧版控制台目录：`frontend/`
- 保留入口：`/legacy`
- 用途：迁移期兜底，不中断已有操作流程

当前策略：

- 新版前端优先承接主控制台和结果查看入口
- 旧版前端暂时保留部分历史操作表单
- 后续按模块逐步把旧入口迁到 `web/`

## 关键接口

- `GET /capabilities`
- `GET /projects`
- `GET /jobs`
- `GET /provider-usage`
- `GET /model-stage-plan`
- `GET /adaptation-packs`
- `GET /adaptation-packs/{pack_name}/latest-result`
  - 返回 pack 最近一次结果对应的明确 `job_id`
  - 优先给出 `artifacts/job_<id>/...` 的权威链接
  - 共享 `latest_result.md` / `latest_validation.md` 仅作为共享副本

## 目录增量

- `web/package.json`
- `web/src/App.tsx`
- `web/src/styles.css`
- `start_web.bat`
- `build_web.bat`

## 依赖管理

### Python

继续由根目录 `requirements.txt` 管理。

### Node

由 `web/package.json` 和 `web/package-lock.json` 管理：

- `react`
- `react-dom`
- `vite`
- `typescript`
- `@vitejs/plugin-react-swc`
- `@types/react`
- `@types/react-dom`

## 启动方式

### 开发

1. 启动后端：`powershell -ExecutionPolicy Bypass -File .\run_backend.ps1`
2. 启动前端：`start_web.bat`
3. 打开：`http://127.0.0.1:5173`

### 构建后由后端直出

1. 执行：`build_web.bat`
2. 启动后端
3. 打开：`http://127.0.0.1:8000`

## 价值

- 前端代码不再和后端模板耦合，便于持续迭代
- 控制台可以直接看到任务进度、结果摘要和关键产物
- 适配包结果默认回到 job 级证据，降低共享副本误引用风险
