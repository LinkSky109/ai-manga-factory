# 原文获取工具调研

更新时间：2026-03-18

这份调研只看“合法原文接入”相关能力，不看盗版站定向爬虫。

结构化底账已经落在：

- `data/reference/source_ingestion_registry.json`

后续脚本可通过：

- `shared/source_tool_catalog.py`

直接读取。

## 结论

当前最值得保留的外部能力不是“某一个万能爬虫”，而是一套分层组合：

1. 静态正文提取：`Trafilatura`
2. 登录态和动态阅读页：`Playwright Python`
3. 人工先打开正版页面再保存：`SingleFile`
4. 单页正文兜底抽取：`readability-lxml`
5. 复杂站点人机协同：`browser-use`

## 推荐组合

### 1. 静态公开正文页

推荐：

- `Trafilatura`
- `collect_source_text.py`

原因：

- `Trafilatura` 同时支持在线 URL 和离线 HTML
- 能输出 `txt / markdown / json`
- 适合把普通网页小说正文清洗成稳定文本

### 2. 登录后官方阅读页

推荐：

- `Playwright Python`
- `collect_source_text.py`

原因：

- 官方文档明确支持把认证状态保存为 `state.json`
- 适合先走一次登录，然后复用登录态抓章节 HTML
- 这比直接匿名请求更接近正版站点现实

### 3. 人工先打开页面再采集

推荐：

- `SingleFile`
- `collect_source_text.py`

原因：

- 这是当前最稳的正版阅读页落地方式
- 用户在浏览器里正常打开章节后，保存成单文件 HTML
- 项目已经能直接导入逐章 HTML

### 4. 复杂站点和人机协同

推荐：

- `browser-use`
- `SingleFile`
- `collect_source_text.py`

原因：

- 对验证码、复杂交互、站内多跳转，纯脚本经常不稳
- `browser-use` 更适合把“人工接管”和“浏览器代理”揉到一起
- 但它更重，不建议当第一选择

## 当前项目里的落地判断

对 AI 漫剧工厂来说，当前最现实的扩展顺序是：

1. `Playwright` 认证导出脚本已落地
2. 下一步补 `Trafilatura` HTML 清洗适配器
3. 继续保留 `SingleFile` 作为人工回退链路
4. 复杂站点再考虑 `browser-use`
