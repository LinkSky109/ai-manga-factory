# 小说原文导入工具

这个工具负责把小说原文整理进适配包的 `source/` 目录，供后面的章节摘要生成、分镜和视频流程直接使用。

只导入你有权使用的内容，例如：

- 你自己持有或已获授权的 `txt` / `md` / `json`
- 官方平台允许访问和处理的章节页面
- 你自己整理的逐章梗概
- 你在浏览器中合法访问后保存下来的章节 HTML

## 输出位置

执行后会写入：

- `adaptations/<pack_name>/source/chapters/chapter_0001.md`
- `adaptations/<pack_name>/source/source_manifest.json`
- `adaptations/<pack_name>/reports/source_collection_report.md`

## 适用入口

### 1. 导入单个 txt 或 md

适合整本小说在一个文件里，并且文本里已经有明显的章节标题。

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\collect_source_text.py --pack-name dpcq_ch1_20 --source-file E:\novels\dpcq.txt --chapter-start 1 --chapter-end 20 --overwrite
```

如果章节标题格式比较特殊，可以自定义正则：

```powershell
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\collect_source_text.py --pack-name dpcq_ch1_20 --source-file E:\novels\dpcq.md --chapter-pattern "(?m)^##\\s*(第.+章.*)$" --overwrite
```

### 2. 导入逐章目录

适合每章一个文件，也支持浏览器保存下来的逐章 HTML。

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\collect_source_text.py --pack-name dpcq_ch1_20 --source-dir E:\novels\dpcq_chapters --chapter-start 1 --chapter-end 20 --overwrite
```

目录里的文件建议类似：

- `chapter_0001.md`
- `第1章 斗之力三段.txt`
- `01-xiao-yan.html`

脚本会优先从文件名或标题里识别章节号；识别不到时，按排序顺序递增补号。

### 3. 按链接清单采集

适合你已经有可合法使用的章节 URL 列表。

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\collect_source_text.py --pack-name dpcq_ch1_20 --url-manifest E:\novels\source_urls.json --overwrite
```

`source_urls.json` 结构现在推荐用：

```json
[
  {
    "chapter": 1,
    "title": "第1章",
    "catalog_title": "第1章 示例标题",
    "match_aliases": [
      "第1章 示例标题",
      "示例标题"
    ],
    "match_keywords": [
      "示例标题"
    ],
    "url": "https://example.com/chapter-1",
    "wait_selector": "body",
    "content_selector": "article",
    "title_selector": "h1"
  }
]
```

### 4. 从目录页抓链接

适合你已经有目录页 HTML，想先自动补出 `source_urls.json`。

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\build_source_url_manifest.py --pack-name dpcq_ch1_20 --toc-file E:\novels\dpcq_catalog.html --base-url "https://www.qidian.com" --merge-existing --force
```

## 正版站点需要登录时怎么办

像起点这类官方站点，匿名请求通常只会拿到校验壳，不是正文。现在项目里已经准备好两条链：

### 方式 A：Playwright 登录后抓取

先保存登录态：

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\playwright_source_capture.py login --pack-name dpcq_ch1_20 --config-file .\adaptations\dpcq_ch1_20\source\playwright_capture.template.json
```

再跑一键流水线：

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\run_source_ingestion_pipeline.py --pack-name dpcq_ch1_20 --toc-file E:\novels\dpcq_catalog.html --base-url "https://www.qidian.com"
```

### 方式 B：浏览器保存 HTML 后离线导入

如果站点的人机校验太复杂，最稳的办法是：

1. 在浏览器中打开你有权访问的章节页
2. 每章保存成 `.html`
3. 放到一个目录里
4. 再用 `--source-dir` 导入

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\collect_source_text.py --pack-name dpcq_ch1_20 --source-dir E:\novels\dpcq_saved_html --chapter-start 1 --chapter-end 20 --overwrite
```

## 导入完成后怎么用

原文导入完成后，直接重生成章节摘要：

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\generate_chapter_briefs.py --pack-name dpcq_ch1_20 --chapter-start 1 --chapter-end 20 --force
```

现在 `generate_chapter_briefs.py` 会优先读取：

- `adaptations/dpcq_ch1_20/source/chapters/`

有逐章原文时，摘要按原文生成；没有原文时，才回退到标题和补充说明驱动模式。
