# AI Manga Factory 服务器部署说明

## 目标

当前项目已支持通过 Docker 在 Linux 服务器上部署，不再依赖 Win11 本地环境。

默认部署形态：

- 后端：FastAPI + Uvicorn
- 前端：Vite 构建产物，由后端直接提供
- 运行数据：挂载到容器外部目录

## 前置条件

服务器建议：

- Linux x86_64
- Docker 24+
- Docker Compose v2
- 至少 2 CPU / 4 GB 内存

如果任务里会生成图片、音频、视频，建议额外准备更高磁盘空间。

## 快速开始

1. 复制环境变量模板：

```bash
cp .env.example .env
cp secrets/runtime_storage_config.example.json secrets/runtime_storage_config.json
```

2. 按实际环境填写：

- `.env`
- `secrets/runtime_storage_config.json`

3. 启动：

```bash
./deploy.sh
```

## 目录说明

部署时重点保留这几个目录：

- `data/`
  - 项目本地数据目录
- `secrets/`
  - 运行配置和密钥目录
- 运行时卷 `ai-manga-factory-runtime`
  - 保存数据库、artifacts、provider usage 等运行产物

## 环境变量

推荐环境变量：

- `AI_MANGA_FACTORY_HOST`
  - 默认 `0.0.0.0`
- `AI_MANGA_FACTORY_PORT`
  - 默认 `8000`
- `AI_MANGA_FACTORY_RUNTIME_DIR`
  - 默认 `/var/lib/ai-manga-factory/runtime`
- `AI_MANGA_FACTORY_RUNTIME_STORAGE_CONFIG`
  - 可选，自定义运行存储配置 JSON 路径

如果使用外部模型或云存储，还需要在 `secrets/` 或环境变量里配置对应密钥。

## Docker Compose 部署

项目根目录已经提供：

- `Dockerfile`
- `docker-compose.yml`
- `.env.example`
- `secrets/runtime_storage_config.example.json`

启动：

```bash
docker compose up -d --build
```

如果想走最短路径，项目根目录也提供：

```bash
./deploy.sh
```

查看日志：

```bash
docker compose logs -f ai-manga-factory
```

停止：

```bash
docker compose down
```

访问地址：

- 首页：`http://<server-ip>:8000/`
- 旧版控制台：`http://<server-ip>:8000/legacy`

如果使用了反向代理，则访问代理后的域名或端口。

## 直接 Docker 部署

构建镜像：

```bash
docker build -t ai-manga-factory:latest .
```

运行容器：

```bash
docker run -d \
  --name ai-manga-factory \
  -p 8000:8000 \
  -e AI_MANGA_FACTORY_HOST=0.0.0.0 \
  -e AI_MANGA_FACTORY_PORT=8000 \
  -e AI_MANGA_FACTORY_RUNTIME_DIR=/var/lib/ai-manga-factory/runtime \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/secrets:/app/secrets \
  -v ai-manga-factory-runtime:/var/lib/ai-manga-factory/runtime \
  ai-manga-factory:latest
```

## 非 Docker 方式

如果服务器不使用 Docker，也可以直接运行：

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
cd web
npm install
npm run build
cd ..
AI_MANGA_FACTORY_HOST=0.0.0.0 AI_MANGA_FACTORY_PORT=8000 ./start.sh backend
```

注意：

- Linux 下使用 `build_web.sh` 和 `start.sh`
- `build_web.sh`、`start_web.sh` 已改成优先使用系统 `npm`，不再写死 Windows `npm.cmd`

## 反向代理建议

生产环境建议放在 Nginx 或 Caddy 后面：

- 反代到 `127.0.0.1:8000`
- 打开 gzip/br 压缩
- 为 `/app/assets/` 设置静态缓存
- 上传限制按媒体产物大小调整

项目已经提供示例：

- Nginx: `deploy/nginx/ai-manga-factory.conf`
- Caddy: `deploy/caddy/Caddyfile`

如果要走 HTTPS，建议由 Nginx/Caddy 负责证书终止。

## 生产环境建议

- 使用单独磁盘或挂载目录保存运行时数据
- 把 `secrets/` 放进服务器私有目录，不要提交真实密钥
- 通过 `.env` 或 Secret Manager 注入 `ARK_API_KEY`
- 监控：
  - 容器存活
  - Compose healthcheck 状态
  - 磁盘占用
  - `/health`
  - 运行时目录增长速度

## 当前限制

- 依赖真实模型服务时，服务器仍需具备对应网络访问能力
- 如果未来要上更严格的 artifact 预览接口，建议再补统一 `/artifact-previews/*` API，而不是只依赖静态文件 + 前端 viewer
