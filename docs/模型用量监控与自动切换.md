# 模型用量监控与自动切换

AI 漫剧工厂会把模型调用用量和最近错误持久化到：

- `E:\work\project-manager\workhome\projects\ai-manga-factory\data\provider_usage\model_budget_config.json`
- `E:\work\project-manager\workhome\projects\ai-manga-factory\data\provider_usage\usage_ledger.json`

## 作用

1. 记录文本、图片、视频模型的调用量
2. 当预计用量逼近阈值时，自动切换到后备模型
3. 当供应商返回欠费、额度不足、限流等错误时，临时封禁当前模型并尝试下一个模型
4. 在前端首页展示当前模型、用量比例、最近切换和最近错误

## 当前基线

当前默认基线按“公开可用的 API 模型”配置：

- 文本主模型：`doubao-seed-2-0-pro-260215`
- 文本后备：`doubao-seed-2-0-lite-260215` -> `doubao-seed-2-0-mini-260215` -> `doubao-seed-1-6-251015`
- 图片主模型：`doubao-seedream-5-0-260128`
- 图片后备：`doubao-seedream-5-0-lite-260128` -> `doubao-seedream-4-5-251128` -> `Doubao-Seedream-4.5`
- 视频主模型：`doubao-seedance-1-5-pro-251215`
- 视频后备：`doubao-seedance-1-0-pro-250528`

说明：

- `Seedance 2.0` 在火山当前公开资料里仍以体验中心能力为主，不作为当前 API 生产基线。
- 因此这套配置里，视频仍以 `1.5 Pro` 为默认主模型，避免把生产链切到尚未正式开放的 API 能力上。
- 用户已开通并纳入配置的模型都会保留为备选；系统不会只保留一个后备模型，而是会按 `priority` 顺序依次尝试。

## 当前统计方式

- 文本模型：按本地估算 `tokens` 统计
- 图片模型：按调用次数统计
- 视频模型：按调用次数统计

## 关键配置

每个模型都可以配置：

- `priority`：优先级，数字越小越先用
- `budget_limit`：本地预算上限
- `enabled`：是否启用

每个供应商可以配置：

- `warning_ratio`：超过这个比例后前端显示预警
- `switch_ratio`：超过这个比例后优先切到后备模型
- `quota_error_cooldown_minutes`：出现额度或欠费错误后的临时封禁时长

## 接口

后端提供：

- `GET /provider-usage`

返回内容包含：

- 当前统计周期
- 配置文件路径
- 账本文件路径
- 文本 / 图片 / 视频三类能力的当前激活模型
- 每个模型的累计用量、成功失败次数、最近错误、封禁状态和最近切换事件
