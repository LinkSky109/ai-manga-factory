# 资产锁驱动的分镜与音频链路

## 目标

把 pack 级资产锁从“提示词附加信息”提升为章节事实源、分镜蓝图、音频计划和 QA 门禁的共同基线。

## 输入

- `adaptations/<pack>/asset_lock.json`
- `adaptations/<pack>/assets/characters/character_cards.json`
- `adaptations/<pack>/assets/scenes/scene_cards.json`
- 章节 brief
- 原文节选

## 章节 JSON 流转

1. `story_grounding.json`
   - 章节事实源
   - 提取真实角色、场景基线、世界规则、可复用台词、有效旁白候选
2. `storyboard_blueprint.json`
   - 内容驱动的镜头蓝图
   - 决定 `target_duration_seconds`、`shot_count`、`keyframe_count`
   - 决定每个 shot 的 `speaker`、`present_characters`、`dialogue`、`narration`
3. `storyboard.json`
   - 最终分镜表
   - `对白角色` 与 `出镜角色` 只允许真实 canonical character
4. `audio_plan.json`
   - 基于真实角色分镜生成配音计划
   - 产出 `dialogue_tracks`、`narration_tracks`、`voice_tracks`
   - 产出 `mix_profile`、`bus`、`priority`、`duck_target`

## 标准化资产卡

### character_cards.json

建议字段：

- `character_id`
- `name`
- `display_name`
- `voice_id`
- `aliases`
- `fixed_prompt`
- `dramatic_role`
- `visual_traits`
- `asset_status`
- `reference_assets`
- `notes`

### scene_cards.json

建议字段：

- `scene_id`
- `name`
- `baseline_prompt`
- `asset_status`
- `reference_assets`
- `lighting_notes`
- `material_notes`
- `camera_guardrails`
- `notes`

## QA 门禁

启用资产锁的 pack 必须满足：

- 分镜中不再出现 `主角 / 同伴 / 对手 / 旁白` 这类泛槽位词
- `出镜角色` 不能为空，且可归一到真实角色
- `audio_plan` 中不存在 generic `canonical_character` fallback
- 资产锁与资产卡引用路径有效
- 关键帧 prompt 命中当前镜头角色的固定特征

## 验收建议

全流程默认先跑最小单元，例如单章 `1-1`：

- 先看 `story_grounding.json`
- 再看 `storyboard_blueprint.json`
- 再看 `storyboard.json`
- 再看 `audio_plan.json`
- 最后核对 `qa_report.md`、`qa_snapshot.json`、`manifest.json`
## 2026-03-30 增补

### story_grounding 质量目标

`story_grounding.json` 现在不仅记录角色名，还要尽量给出：

- `cleaned_source_lines`
- `scene_anchors`
- `conflict_points`
- `dialogue_candidates_detailed`
- `character_relationships`

这些字段是后续 blueprint 去模板化和减少废旁白的直接输入。

### 可审阅资产卡

资产卡默认补齐以下审阅字段：

- `asset_status_detail`
- `review_status`
- `approval_notes`
- `owner`
- `review_checklist`
- `source_evidence`
- `last_verified_job_id`
- `usage_scope`

### pack 章节时长计划

适配包运行时支持：

- `pack.json.default_target_duration_seconds`
- `chapter_briefs.json[*].target_duration_seconds`

系统会在内部整理成 `chapter_duration_plan`，并优先驱动 `storyboard_blueprint.json.target_duration_seconds`。
