# 运行时数据区与云存储

## 当前推荐

- 运行时数据根目录默认放到 `OneDrive/CodexRuntime/ai-manga-factory`
- 项目内 `data/reference/` 只保留静态参考资料
- 运行时产物不再默认写回仓库目录

这样做的好处是：

- 大体积产物、数据库、会话、日志不会污染 Git 工作区
- 机器重装或切目录后，运行时产物还在同步盘里
- 后续可以继续接对象存储，而不是把业务代码绑死在某个本地盘符

## 路径规则

- 静态参考数据：`<project>/data/reference/`
- 运行时数据：`AI_MANGA_FACTORY_RUNTIME_DIR` 或 `secrets/runtime_storage_config.json` 指定的 `runtime_root`
- 自动默认值：优先 `OneDrive/CodexRuntime/ai-manga-factory`，否则回退到 `<project>/data_runtime/`

## 配置文件

本地配置文件：

- `secrets/runtime_storage_config.json`

示例结构：

```json
{
  "runtime_root": "C:/Users/Administrator/OneDrive/CodexRuntime/ai-manga-factory",
  "runtime_provider": {
    "type": "sync_folder",
    "name": "onedrive",
    "root_path": "C:/Users/Administrator/OneDrive/CodexRuntime/ai-manga-factory"
  },
  "remote_sync": {
    "enabled": false,
    "provider": "aliyun_oss",
    "aliyun_oss": {
      "endpoint": "https://oss-cn-hangzhou.aliyuncs.com",
      "bucket": "your-bucket",
      "prefix": "ai-manga-factory/runtime",
      "access_key_id_env": "AI_MANGA_FACTORY_OSS_ACCESS_KEY_ID",
      "access_key_secret_env": "AI_MANGA_FACTORY_OSS_ACCESS_KEY_SECRET"
    },
    "baidu_pan": {
      "status": "planned"
    },
    "quark_pan": {
      "status": "planned"
    },
    "aliyundrive": {
      "status": "planned"
    }
  }
}
```

## 已支持

### 1. 同步盘目录模式

适合：

- OneDrive
- 百度网盘客户端同步目录
- 夸克网盘客户端同步目录
- 阿里云盘客户端同步目录

本质上是把 `runtime_root` 直接放在同步目录里，业务代码仍然写本地文件系统，但文件会被网盘客户端同步走。

### 2. 阿里云 OSS

适合：

- 需要官方、稳定、脚本化的远端存储
- 需要后续扩展生命周期管理、归档、CDN、跨机访问

同步命令：

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\sync_runtime_storage.py
```

安装 OSS 依赖：

```powershell
E:\work\.venvs\ai-manga-factory\Scripts\python.exe -m pip install -r .\requirements-storage.txt
```

## 当前没有直接接入的网盘 API

下面这些我已经留了配置入口，但当前没有在生产里直接接它们的 HTTP API：

- 百度网盘
- 夸克网盘
- 阿里云盘

原因很简单：这些产品的“通用文件存储 API”没有像 OSS 一样稳定、公开、适合当生产后端。当前更稳的做法是：

- 网盘：走同步目录模式
- 正式远端存储：走 OSS

## 迁移现有运行时数据

```powershell
cd E:\work\project-manager\workhome\projects\ai-manga-factory
E:\work\.venvs\ai-manga-factory\Scripts\python.exe .\scripts\migrate_runtime_data.py
```

它会把这些内容复制到新的运行时目录：

- `artifacts/`
- `provider_usage/`
- `requirements/`
- `source_sessions/`
- `platform.db`
- `backend.log`
- `backend-error.log`
- `provider_test_i2v.mp4`
